Model View Controller- MVC

Es un patrón de diseño en software, que permite la separación de obligaciones de cada pieza de código en tu aplicación web.
Enfatiz la separación de la lógica de programación y lo que se muestra en pantalla.

-El model:
Encargado de los datos (desde una base de datos) y de la lógica para mostrar esos datos.
Ejemplo: Un usuario quiere ver la sección de productos, el modelo se encargará de realizar esa consulta en la base de datos.

-El view:
Se encarga de lo que se ve en pantalla (html)
Ejemplo: Si el modelo hace la consulta a la base de datos para los productos, es la vista la que muestra esos resultados.

-El controller:
Es el que se comunica entre el modelo y la vista, antes de que el modelo consulte a la base de datos, este (controller) se encarga de mandarlo a llamar (modelo), y una vez que el modelo tiene los resultados de la consulta, el controller se encarga de pasarlos a la vista.

-El router:
Encargado de registrar todas las url's o endpoints que la aplicacións soporta.
Ejemplo: si el usuario accede a /productos, el router llama un controlador, que se comunica con el modelo para obtener los datos que son pasados hacia la vista para ser mostrados

DIAGRAMA 

Router /Productos viene enlazado a Controller
            |<------------
MODEL       |        CONTROLLER         |       VISTA
   -------->|             ------------->|

El router llama al controlador de la url usada por el usuario
El controlador le pasa la consulta al model
El model resuelve la consulta y le pasa los datos al controller
EL controller le pasa los resultados recibidos del model al vew para ser mostrados.
